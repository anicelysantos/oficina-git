# 🧪 Oficina Git e GitHub

Bem-vindo(a)!
Nesta atividade você vai aprender, na prática, como usar o Git e o GitHub.

---

## 🎯 Objetivo

Você vai aprender a:

* Fazer um fork de um repositório
* Editar arquivos
* Criar commits
* Criar uma branch
* Abrir um Pull Request
* Receber feedback automático

---

## 🔁 Fluxo da atividade

```
Fork → Editar → Commit → Branch → Pull Request → Validação
```

---

## 🧩 PASSO A PASSO

### 1️⃣ Fazer um Fork

1. Clique no botão **Fork** no topo da página
2. Isso criará uma cópia do repositório na sua conta

---

### 2️⃣ Criar uma Branch

1. Clique no botão **main**
2. Digite um nome para sua branch, por exemplo:

```
minha-contribuicao
```

3. Clique em **Create branch**

---

### 3️⃣ Desafio 1 — Adicionar seu nome

1. Abra o arquivo:

```
desafio/01-commit/alunos.md
```

2. Clique em ✏️ (editar)

3. Adicione seu nome em uma nova linha

4. Clique em **Commit changes**

---

### 4️⃣ Desafio 2 — Editar arquivo

1. Acesse:

```
desafio/02-branch/arquivo.txt
```

2. Faça qualquer alteração no conteúdo

3. Clique em **Commit changes**

---

### 5️⃣ Desafio 3 — Responder pergunta

1. Abra:

```
desafio/03-pull-request/resposta.md
```

2. Responda à pergunta proposta

3. Clique em **Commit changes**

---

### 6️⃣ Criar Pull Request

1. Clique em **Contribute**
2. Clique em **Open Pull Request**
3. Clique em **Create Pull Request**

---

## 🤖 Avaliação automática

Após criar o Pull Request:

* O sistema irá verificar suas respostas automaticamente
* Você verá um resultado:

### ✅ Sucesso

Tudo certo! Você concluiu a atividade.

### ❌ Erro

Algo está faltando. Verifique:

* Você adicionou seu nome?
* Você respondeu a pergunta?
* Você fez commit das alterações?

👉 Vá na aba **Checks** para ver o erro detalhado.

---

## ⚠️ Problemas comuns

### ❌ Não aparece botão de Pull Request

👉 Você não fez commit ainda

---

### ❌ Deu erro na avaliação

👉 Você não completou todos os desafios

---

### ❌ Editei mas não funcionou

👉 Verifique se editou os arquivos corretos

---

## 💡 Dica importante

Você pode corrigir seu Pull Request:

* edite os arquivos novamente
* faça commit
* o sistema irá reavaliar automaticamente

---

## 🎉 Parabéns!

Ao finalizar, você terá praticado:

* Controle de versão
* Trabalho com branches
* Colaboração com Pull Request
* Uso do GitHub na prática

---

## 🚀 Extra (opcional)

Se estivesse usando Git localmente, o fluxo seria:

```
git clone
git checkout -b minha-branch
git add .
git commit -m "minha alteração"
git push
```

Depois abrir o Pull Request no GitHub.
